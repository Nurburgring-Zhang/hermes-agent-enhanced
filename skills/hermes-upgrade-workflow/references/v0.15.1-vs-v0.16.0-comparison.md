## 升级路径

### 网络受限环境的升级方法（WSL + GFW）

```bash
# 方案A: 直接git fetch（需可连通GitHub）
git fetch --depth=1 --no-tags origin +refs/heads/main:refs/remotes/origin/main

# 方案B: 用Windows下载tarball（WSL内git不可达时）
# 在Windows cmd中:
curl -L --connect-timeout 15 "https://github.com/NousResearch/hermes-agent/archive/refs/tags/v2026.6.5.tar.gz" --max-time 300 -o C:\Users\Administrator\hermes-v0.16.0.tar.gz

# 如果下载不完全（~88KB/s网络限速），优先获取核心文件:
# 检查tar中是否有 run_agent.py + conversation_loop.py + hermes_cli/__init__.py
tar tzf hermes-v0.16.0.tar.gz 2>/dev/null | grep -E '(run_agent|conversation_loop|hermes_cli/__init__)'

# 方案C: 先fetch一部分再reset
git fetch --depth=1  # 可能超时，可重试
git reset --hard origin/main  # 覆盖本地（前提是已备份自定义文件）
```

### 升级后必须做的3个注入

每次升级后需要重新注入的3个核心修改：

```bash
# 1. conversation_loop.py — PRE hook
# 在 effective_system = active_system_prompt or "" 之后插入:
# try: import sys; ... from agent_enhancement_manager import safe_hook_pre_conversation

# 2. run_agent.py — POST hook
# 在 return run_conversation(...) 改为:
# result = run_conversation(...)
# try: ... safe_hook_post_conversation(...)
# return result

# 3. hermes_cli/__init__.py — 启动加载
# 在 import os 之前插入双审加载脚本调用

# 4. model_tools.py — 进程级强制加载（2026-06-12新增）
# 在文件末尾添加 _force_init_all_enhancements()
```
