# 灰白稿故事版方法

来源: GPT-Image2分镜故事版生视频实战方法

## 为什么要用灰白稿

实景画面的故事版会导致视频模型判断混乱——无法区分"这是参考图"和"这是最终结果"。
灰白稿让视频模型专注于理解构图、运镜和动作逻辑。

## 每镜头独立策略

不要把所有分镜放在一张图里——信息量过大会导致模糊。
**每镜头一个独立灰白稿**，具体标注：
- 运镜方向
- 画面描述
- 时长
- 台词
- 人物表情动作
- 环境和音效

## GPT-Image2 → Seedance2.0 管道

### Step 1: 文字分镜
建立电影级专家角色→写出场景的动作调度（角色位置、运动轨迹、镜头角度）

### Step 2: GPT Image 2 → 调度分镜图
生成带运动轨迹的previs调度图，推荐用火柴人（方便后期复用角色）

### Step 3: Seedance 2.0 → 视频
提示词格式:
```
这是角色xx设定图 {{Portrait 1}}，
严格按照这张图 {{Portrait 2}} 中的N镜头PREVIS演绎动作。
下面是文字版动作分镜：[粘贴文字分镜]
```

## 角色引用机制
- 通过 @图片 引用角色资产到GPT-Image2
- 多个角色就@对应的角色资产
