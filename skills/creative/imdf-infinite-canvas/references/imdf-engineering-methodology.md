# 生产体系工程规范（IMDF实践版）

## 从需求到交付的完整流程

遵循格林主人的"先调研再出计划再实施"工作模式：

### Phase 1: 需求定义
- 明确边界、范围、核心能力
- 确定"世界顶级"的参照标准
- 输出: 需求说明书

### Phase 2: 全局信息采集
- 全网搜集相关项目/文章(目标63+, 覆盖GitHub/论文/商业产品/微信公众号)
- 每个项目记录: 核心架构/参数/限制/许可证/Stars
- 竞品对比: 我们vs他们的差异点
- 输出: 调研报告(500+行, 结构化)

### Phase 3: 架构设计
- 融合多个世界级方案的最强特性(不选一弃一)
- 每引擎有明确的"融合方案": 方案A × 方案B × 方案C
- 对比世界最顶级方案(InvokeAI/Toonflow/Claude Design/html-video/Frontend Slides)
- 输出: 架构设计文档

### Phase 4: 开发计划
- 按WBS拆解为P0/P1/P2/P3
- 每个模块有独立的验收标准
- 测试先行: 写代码前先写测试

### Phase 5: 实现 + TDD
- Python: 类型注解、dataclass、pytest
- 每个public函数有docstring
- 函数≤60行, 文件≤500行(生产引擎可放宽)
- 改前备份到 /mnt/d/Hermes/备份/

### Phase 6: 集成验证
- 单元测试 41/41
- 端到端全链路验证(6引擎全部过)
- Web UI可用性验证(HTTP 200)

## 世界级系统设计原则
1. 不重新发明轮子: 先搜集63+项目再融合
2. 不选一弃一: 融合多个方案的最强特性
3. 每层都有质量门禁: Checkpoint + Reviewer + Quality Gate
4. 有fallback: 主引擎失败自动降级不崩
5. 真实可运行: 拒绝stub/mock/占位符
6. Agent驱动: 不是工具,是自主生产系统
