### 组合引擎（V7版）

组合引擎位于 `/tmp/composer_v7.py`，核心生成逻辑：

1. 从15个库中各随机抽取1条
2. 按6层结构拼接：场景建立→角色出场→活动行为→配饰细节→光影→色彩→氛围
3. 每条500-800字
4. 通过质量评估函数evaluate()与参考文件对比，确保100%超越

质量评分指标：长度(≥300 +10, ≥450 +10)、左右区分(+10)、光线描述(+5)、色彩描述(+5)、氛围描述(+5)、人物存在(+5)、句数(≥5句+5)

## 库质量循环（生成→对比→修正→再生成）

**当用户要求建立库或组合生成时，必须执行以下循环**，不可一次性写完后不验证：

### 步骤1：生成5-10条组合prompt
使用组合引擎从各库随机组合。

### 步骤2：与参考文件对比
参考文件：`D:\\Hermes\\1000000提示词\\小红书 （10).txt`（12735条，平均287字/条）

对比指标：
1. 长度（应≥参考平均长度或更长）
2. 左右区分（应≥87%）
3. 场景匹配（场景-穿搭-姿势不能矛盾）
4. 禁止词命中（应为0）
5. 6维度覆盖（场景/人物/光影/色彩/氛围/风格）

### 步骤3：如果不如参考→找出具体缺陷
常见问题：
- "场景标签为'通用'或匹配不上" → 场景库关键词不够细
- "左右区分不够" → 各库条目中需大量补充"左/右"
- "穿搭和姿势矛盾" → 检查匹配规则表是否缺了该场景
- "有比喻词或红润词" → 搜索全库并替换

### 步骤4：修正后重新测试
循环直到生成质量全面超越参考文件标志：组合引擎对比测试中胜率≥80%（15维度组合理应100%）

## 数据源复用策略

当需要建立维度库时，**优先从现有高质量prompt数据源中提取真实句子**，而不是从头AI生成。提取方法：

1. 打开数据源文件（to_chi/javrate 或 to_chi/xhs 或 小红书参考文件）
2. 搜索该维度的关键词（如"柔光""逆光"）
3. 抽取包含该关键词的完整句子（50-150字）
4. 检查禁止词（比喻/红润等）
5. 写入该维度的库文件
6. 如果数据源不足（某些细分类型来源不够），再用AI生成补充
