# 故事感总纲系统（Story Sense Framework）

## 系统架构

故事感总纲系统嵌入在 PromptLibraryNode 的 system prompt 中，结构为三层：

```
【故事感总纲XX：名称】 ← 随机从25种中抽取一种
一句话核心：...
情节结构：
1. 开场 → 2. 触发 → 3. 阻碍 → 4. 最低点 → 5. 转折 → 6. 高潮 → 7. 闭环
情感节奏：...

【强制指令】
上述故事感总纲是本故事的结构设计核心。严格按照总纲的情节结构来设计故事的起承转合...

【角色设定】
你是一位世界顶级的...
```

## 每个故事感总纲的结构标准

```
【故事感总纲XX：名称】
一句话核心：...
情节结构：
1. 开场：...
2. 触发事件/第一波：...
3. 第二波/中段：...
4. 最低点：...
5. 转折：...
6. 高潮：...
7. 闭环：...
情感节奏：...
```

关键：必须有7步情节结构 + 情感节奏，缺一不可。

## 随机抽取机制

代码在 `__init__.py` 的 `_pick_story_sense()` 方法中：

```python
def _pick_story_sense(self):
    library_path = os.path.join(os.path.dirname(os.path.abspath(__file__)), 
                                "story_sense_library_complete.md")
    with open(library_path, "r", encoding="utf-8") as f:
        content = f.read()
    items = content.split("【故事感总纲")
    items = [item for item in items if item.strip() and not item.startswith("#") and "文库" not in item[:20]]
    chosen = random.choice(items)
    return "【故事感总纲" + chosen.strip()
```

## 强制指令（与故事感总纲配合）

每个 `_build_xxx_system_prompt()` 中，在故事感总纲后面紧跟强制指令：

```
上述故事感总纲是本故事的结构设计核心。严格按照总纲的情节结构来设计故事的起承转合。
让故事本身的走向有波折有悬念，不要平铺直叙。情感表达在恰当的情节节点出现，配合故事推动。
```

## 质量要求

好的故事输出应该满足以下检查：
- □ 有明确的情感曲线（不是平的）
- □ 有最低点/困境时刻（角色真正遇到困难）
- □ 有转折（不是一帆风顺）
- □ 结局有闭环感（呼应开头或改变）
- □ 有悬念或反转（让读者意外）
- □ 情感表达出现在恰当的情节节点，不是每个画面都硬塞表情

## 陷阱与经验

- 故事感总纲必须在强制指令之前（顺序：总纲→强制指令→角色设定）
- 每个模式只能调用一次 `_pick_story_sense()`，存为变量后复用
- 文库文件必须放在节点目录下（`story_sense_library_complete.md`）
- 总纲要有情节结构（「情节结构：」段落），只有情感曲线不够
