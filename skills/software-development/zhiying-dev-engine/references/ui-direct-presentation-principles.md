# UI设计原则：常用功能直接呈现

## 核心原则

用户不接受"功能都有但藏在菜单里"。每次UI设计必须遵循：

### 首页默认视图（所见即所得）

打开页面后直接看到：
1. **4个KPI指标卡** — 今日生产量/待审核任务/在线人数/系统状态
2. **8个快捷操作大按钮** — 上传数据/开始标注/执行工作流/查看看板/创建任务/邀请成员/交付数据/统计分析
3. **最近任务列表** — 带进度条和状态的实时列表
4. **快速标注工具** — 拖拽上传+AI预标注入口
5. **左侧固定导航** — 常用功能区（今日概览/我的任务/数据集/标注工具/工作流）在上，完整菜单在下

### 交互规范

- 新用户打开页面，5秒内应能说出"这个平台至少能做什么"
- 核心操作（上传/标注/执行）必须在首页有直接入口，不需要翻菜单
- 每个业务页面顶部有搜索+操作按钮栏，底部有分页器
- 模态框用于临时操作（预览/确认/配置），不跳转页面
- 状态栏始终可见，显示核心系统指标

### 对标基准

Scale AI / Labelbox / Supervisely 的首页布局：
- 左侧固定导航
- 首页直接显示指标卡+最近活动
- 顶栏搜索+通知+用户

IMDF的v3设计对标这些平台，增加了：
- 更多快捷操作按钮（8个 vs 通常的4-5个）
- 首页内置快速标注小工具
- 状态栏显示更多运维指标

### 验证标准

1. 打开首页后，所有常用功能入口应在第一个视口内可见（无需滚动）
2. 点击任何一个快捷按钮，应在2步内完成核心操作
3. 任何页面可通过左侧导航在2次点击内到达
