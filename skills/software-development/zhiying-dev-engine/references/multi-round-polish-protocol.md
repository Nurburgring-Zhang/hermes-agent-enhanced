# 商用级多轮深度打磨协议 (2026-06-16实战固化)

## 触发条件
用户说"商用级深度打磨"、"多轮打磨"、"继续打磨" → 立即启动此协议，不等指令。

## 多轮结构
每轮打磨有独立目标，递增深度：

### 轮次1: 基础验收
- 基础设施(服务/认证/限流/审计) → 6项
- 核心业务(API/数据/AI) → 8项  
- 前端页面(所有导航项) → 8项
- 高级功能(标注/评测/管线) → 4项
- 目标: 25/25全部通过

### 轮次2: 运行时验证
- R10 幂等性: SHA256键生成+去重验证
- R11 Checkpoint: >10步自动触发
- R12 质量门禁: 5道门全部就绪
- R13 模型多样性: 不同模型家族验证
- R15 降级策略: 5级降级可用

### 轮次3: 边缘情况
- 20并发全200(0.06s)
- 空搜索/超限分页优雅处理
- 错误输入422/优雅降级
- 响应<10ms
- 50连续请求无泄漏

### 轮次4: 联动测试
- 12步任务全链路: 幂等+Checkpoint+降级联动
- 三机制(R10+R11+R15)同时验证
- 模拟完整的数据采集流水线

### 轮次5: 极限压力
- 100/100连续请求成功
- 断点保存+恢复
- 50KB大文件9ms响应
- 所有API全部200

## 每轮固定输出
1. 验证结果 (PASS/FAIL/WARN计数)
2. 修复的问题清单
3. 新增的能力

## 深度递增原则
- R1: 覆盖面 — 所有模块的通用验证
- R2: 正确性 — 核心机制的真实运行
- R3: 鲁棒性 — 边界和错误处理
- R4: 一致性 — 多机制联动
- R5: 稳定性 — 压力和持续可用

## 禁止行为
- ❌ 一轮就停 → 说"全部完成"
- ❌ 只测API不测运行时
- ❌ 只测正常路径不测边界
- ❌ 不问用户直接停止
