# PromptLibraryNode V20.5 — 变化追踪与模板格式设计模式（2026-06-01 最终版）

## 核心约束

本会话（2026-06-01）格林主人反复修正了3次，最终确立了以下约束：

### 变化标注规则（最终版）

所有模板中变化标注应遵循：

1. **有变化才输出** — 无变化时整个「变化标注：」段不出现。严禁写「无」。
2. **只写有变化的维度** — 不写无变化维度的占位文字。
3. **中文文字，不用emoji/符号** — 维度名称用中文文字（场景变化/角色变化/服装变化/道具变化/时间变化），不用🔄👤👗📦☀️等符号。
4. **格式** — `变化标注：场景变化——从XX切换到XX；角色变化——XXX` 多维度间用分号分隔。

### 分镜场景/角色特征输出规则

当场景或角色特征变化时，在分镜内输出完整描述（不仅是简短标注）：

- **分镜场景**：场景变化时输出完整场景描述（类似总纲中的场景设定），包含地点、时间、光线、环境氛围、关键道具位置，2-4句
- **角色特征**：角色特征变化时输出完整角色描述（类似总纲中的角色物品设定），包含外貌、服装、表情、状态变化，2-3句
- 两者都变化时各行一行

### 总纲格式规则（2026-06-01 修正版）

所有模板总纲使用统一编号结构，**禁止空行**：

```
【xxx总纲】
1、整体视觉风格：
    内容（4空格缩进）
2、角色物品设定：
    角色1描述
    角色2描述（如有，带4空格缩进）
3、道具/武器：
    待补充。
4、场景设定：
    场景描述（末尾不要句号）
5、氛围与画质标准：
    内容
6、声音设定：
    内容
7、核心叙事设定：
    内容
```

**关键约束：**
- 角色物品设定中多角色换行时，每角色都带4空格缩进（通过`.replace("\n", "\n    ")`实现）
- 角色物品设定末尾不要加句号（避免多角色时句号单独一行）
- 场景设定末尾不要加句号（用户输入可能自带句号导致双重句号）
- 每个字段直接以4空格开始接内容，不要用 IND 变量拼接（避免多级缩进）

### 符号禁令（所有模板）

system prompt中不得出现以下符号（否则AI会模仿到输出中）：

| 禁止 | 原因 |
|------|------|
| `# ` 开头的标题 | AI会在输出中加# |
| `## ` | 同上 |
| `**` | AI会在输出中加**包裹文字 |
| `- ` 开头的列表 | AI会在输出中用-做项目符号 |
| `- **` | 双层符号 |
| `🔴` | AI会输出红色圆点 |
| emoji符号 | AI会输出到变化标注中 |
| `《》`外的`【】`内的emoji | AI会模仿 |

**正确做法：** 所有段标题直接用纯文字（如"角色设定"而非"# 角色设定"），列表项去掉`- `前缀。
