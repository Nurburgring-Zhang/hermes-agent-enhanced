# PromptLibraryNode V19 — 内容密度约束实战记录

格林主人在2026-05-27的V19改造中反复纠正了模板的输出密度问题。以下是实战总结。

## 问题演变

### 第1轮：模板过度细节
AI生成的【总纲】包含"毛密度约200根/cm²""毛细血管""色温4800K""下颌肌群收缩"等无用参数。

**修复：** 在全局约束中加入禁止数值参数的指令。

### 第2轮：矫枉过正，内容太干
AI产出变成"野猪鼻子拱泥土"这种一句话干巴巴的描述。

**修复：** 将"不超过5句话"改为"3-6句话，根据情节调节"，加入"有画面感的叙事语言""有血有肉"等正向引导。

### 第3轮（2026-05-27下午）：统一输出格式导致模式雷同
所有模式共用 `_build_global_context` 的同一套「【通用设定与核心世界观】」模板，输出结构全是总纲→场景→氛围→声音→---→内容。

**修复：** 删除 `_build_global_context` 的全局调用，9个故事板模式+绘本+短剧+儿童4模式各自定义独立的格式。

### 第4轮（2026-05-27晚）：古装风/喜剧风/言情风回归
用户发现古装风/喜剧风/言情风/悬疑风等预设风格不见了，被"电影感"一个自由字符串替代。

**修复：** "画面风格"从自由字符串改为24种预设下拉。

## 关键教训

### 1. 禁止措辞 + 正向引导 = 双向约束

```
❌ 单一约束： "禁止堆细节"（AI理解不了"堆"的边界）
✅ 双约束：   "禁止数值参数和微观细节。用有画面感的叙事语言。"
```

### 2. 数量级约束优于模糊约束

```
❌ 模糊： "适量""不要太多"
✅ 明确： "每个镜头3-6句话""每个板块3-5句话"
```

### 3. 末尾覆盖原则

System prompt中**后面的指令覆盖前面的**。

### 4. 每个模式的输出格式必须与模式名严格匹配

**不要**给所有模式共用同一套输出格式模板。9个故事板模式需要9套完全不同的格式（电影=景别/运镜/转场+时长，广告=卖点/品牌元素/LOGO位置，动画=关键帧/夸张度/特效，漫画=页面布局/格划分/对话框/拟声词...）。

### 5. 预设下拉 > 自由字符串

```
❌ "画面风格": ("STRING", {"default": "电影感"})  → 用户忘记填就是"电影感"
✅ 24种预设下拉 → 用户立刻看到所有选择，不会漏
```

## 最终架构（V19 final）

```
故事板9模式 → 各自独立的 format_templates 字典（9个不同的输出格式）
绘本模式    → 自己独有的4维度格式
短剧模式    → 竖屏+台词+反转的独有内容
儿童4模式   → 各自独立（视频分段/起承转合幕/gif循环/绘本页）
设计8模式   → 各自独立（保留V18的设计质量）
_build_global_context → 改名 _build_design_global_context（仅设计模式使用）
```
