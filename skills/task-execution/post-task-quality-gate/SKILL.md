# Post-Task Quality Gate

## 何时使用
每次任务完成、准备交付给用户之前。必须在最终输出前执行。

## 5项检查

### 1. 输出完整性检查
- 是否有任何声称"已实现"但没有运行证据的功能？
- 代码/配置更改后是否运行了验证？
- 如果涉及多个文件，是否全部确认？

### 2. 运行证据检查
- HTTP API是否有curl输出或HTTP状态码？
- 前端构建是否通过（npm run build）？
- 测试是否通过（pytest）？
- 如果是不可能验证的（如缺GPU），是否明确告知用户？

### 3. 反幻觉检查
- 是否有任何信息不是从实际运行结果得出的？
- 文件路径是否真实存在？
- 声称的功能是否真的可调用？

### 4. 透明度检查
- 如果有未完成的部分，是否明确列出？
- 如果有已知限制，是否告知用户？
- 如果有替代方案，是否给出？

### 5. 三AI互审
- 如果是开发任务，执行AI和监督AI是否都输出了结论？
- 是否有不一致需要用户决策的地方？

## 通过标准
全部5项通过才能交付。如果有任何一项不通过，标记为"QUALITY GATE FAILED"并说明原因。
