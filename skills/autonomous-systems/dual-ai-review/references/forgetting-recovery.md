# 双审"遗忘"模式的恢复策略

## 问题
执行AI在专注执行任务时,会自动跳过预审步骤。这不是恶意的——是工具调用上下文切换太快,AI来不及在每次调用前做"暂停→自检→继续"的元认知步骤。

## 恢复策略

### 策略1: 阶段回溯
当用户指出"AI互审没有执行"时,不要道歉了事。立即:
1. 停止当前任务
2. 回顾最近N步工具调用
3. 每步做一次事后验证
4. 从当前步开始恢复预审

### 策略2: 强制检查点
每完成3次工具调用后,无论前3次有没有做预审,第4次调用前必须暂停并做一次整体回顾。

### 策略3: 自律降级
当delegate_task不可用时,自律审核至少拦截:
- 高风险工具名: delete/remove/rm/drop/shutdown/reboot/format
- 危险模式命令: rm -rf/DROP TABLE/DROP DATABASE/shutdown/> /dev/sda/chmod 777
- 不依赖主动提交,在工具参数中自动检测

### 日志模板
```
[双审恢复] 回溯最近N步:
  步1: {工具名} — 风险: {低/中/高} — 状态: {通过/需审查}
  步2: ...
步N完成后恢复实时预审。
```
