# Nanobot Factory UI架构实战（2026-06-12）

## 覆盖范围
- 首页重写：8卡片导航门户 + 搜索框 + 用户状态显示
- 智影工场13菜单项SPA（新增ML模型/多模态标注/数据集版本/质量中心）
- AIGC工作室（Omnigen工坊+历史画廊+批量生成+进度轮询）
- 工作流编辑器（25节点+3模板+保存加载+执行）
- 三AI互审UI验收

## 关键架构决策

### 为什么首页用纯HTML而不是Vue
- 首页不需要复杂交互，8卡片+导航+状态栏即可
- 减少加载时间，暗色主题与studio.html一致
- 纯原生JS，无框架依赖

### 为什么zhiying.html继续用Vue3
- 已有1560行Vue SPA，13个页面之间做SPA切换
- 重构为组件化架构成本太高（需要Vite+组件拆分）
- 策略：单文件Vue SPA做功能集成，后续再工程化

### 工作流编辑器保持独立
- 与zhiying的CRUD管理功能不同，workflow是创作类工具
- 独立页面=更少的状态管理复杂度
- 通过侧边栏导航保持可达性
