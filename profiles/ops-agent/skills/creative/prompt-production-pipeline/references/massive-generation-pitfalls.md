# Python 批量生成引擎实战踩坑记录

## 会话背景
2026-05-21 使用massive_generator.py从15/16/17/13/18等模板库组合生成20万+条中文prompt。

## 关键教训

### 1. 去重策略（最关键）

**错误做法**（v3.0）：跨文件全局去重 + 用前40字做key。
→ 结果：2000风格×2000穿搭的组合中，前30-40字高度重复，导致每文件只能产出~1600条后就撞光了。

**正确做法**（v4.1）：文件级去重 + 前20-30字做key。
→ 每个文件独立维护一个 `set(line[:30])`，不关心其他文件。
→ 每个文件可产出5000-20000条（取决于开头模板多样性）。

**更激进方案**：如果需要更多，可以用 `line[:20]` 或 `line[:15]` + 方向词前缀（如 `【美式复古】`）做key，可产出3万+条/文件。

### 2. 模板库大小与组合数

- 2000风格 × 2000穿搭 × 2000姿势 × 20开头 = 1.6亿种理论组合
- 但实际去重中，同一模板库内的条目开头相似度远高于预期
- 瓶颈不在组合数，在**开头20-30字的多样性**
- **解决方案**：每个prompt开头强制加入方向词（如"美式复古"、"波西米亚"），确保前20字不重复

### 3. delegate_task子代理的可靠性问题

- `delegate_task` 一次最多3个并行（max_concurrent_children=3）
- 每个子代理生成8条约需2-4分钟（含反复读取/写入文件的开销）
- 父对话如果超时，所有子代理会被杀死 → 产出丢失
- 子代理错误处理：有时会因输出过长被截断而失败（08.txt案例）
- **结论**：delegate_task只适合少量高质的单场景prompt

### 4. cronjob的实际局限

- 9个cronjob理论上45条/5分钟，但cronjob系统不稳定
- 创建的cronjob可能无法触发（`last_run_at` 为null）
- 对同一任务的竞争写入可能导致文件损坏

### 5. 文件组织建议

- 不要把所有prompt放在一个文件（10万条×600字=60MB，打开/搜索太慢）
- 按主题方向分10个文件，每个1-2万条
- 文件命名带主题方向或编号（modern_art_prompts_02.txt）
- 每个文件独立去重，不要跨文件去重

#### 7. 模板库输出的泛光/红润/阴影过度描述问题

**发现**：Python组合引擎从15_风格库/16_穿搭库等模板随机组合后，prompt中充斥大量重复的过度光影比喻：
- 泛着（泛着珍珠般的光泽、泛着柔和的光晕、泛着温润的光）
- 镀上（镀上一层金色、镀上温暖基调）
- 光晕（27884次！平均每7条就有1条带光晕）
- 红润（面部红润、白里透红、红润光泽）
- 倾泻（光线倾泻而下）
- 沉入（沉入温柔的阴影中）
- 投下斑驳（投下斑驳的光影）
- 映红（映红了她的脸和手臂）

**原因**：模板库的原始条目本身就带了大量这类套话描述。组合式生成放大了它们出现的频率，因为多个模板层（光线层+色彩层+氛围层）可能都使用了类似表述。

**解决方案**：生成后的全量清理脚本
```python
# 删除类：完全去除这些套话
DELETE_PHRASES = [
    "在晨光中折射出细碎的光斑",
    "反射出珍珠般的光晕",
    "沉入温柔的阴影中",
    "映红了人物的脸和手臂",
    # ... 详见完整脚本
]

# 替换类：改为更简洁的表述
REPLACE_PAIRS = [
    ("泛着", "带着"),
    ("镀上", "染上"),
    ("光晕", "光"),
    ("红润", "透亮"),
    ("倾泻", "照入"),
    ("投下斑驳", "落下斑驳"),
    # ...
]
```

**清理策略**：
1. 先全量替换（将"泛着""镀上""光晕"等全部替换）
2. 再删除确切的套话长句
3. 最后清理重复标点（"的的"→"的"，"，，"→"，"）

**备份**：清理前自动创建 .bak 备份，随时可恢复。

**脚本位置**：`scripts/clean-lighting-overdescriptions.py`（本技能目录下）

## 6. 质量-速度权衡

| 方法 | 速度 | 质量 | 推荐场景 |
|------|------|------|---------|
| Python组合引擎 | 500条/秒 | ★★★ (模板化) | 大规模填充 |
| AI单条生成 | 1条/30秒 | ★★★★★ (自然流畅) | 精品/特殊场景 |
| 混合策略 | 快 | ★★★★ | Python先批量，AI再精修其中1-2万条 |
