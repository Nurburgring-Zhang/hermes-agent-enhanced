# V3.3 经验引擎: 全自动经验总结 + 跨任务/跨会话复用

## 架构

2026-05-20构建, 解决8个真实GAP后新增。

核心文件: `~/.hermes/evolution_v3/experience_engine.py`

### 三步流程

```
Step 1: 提取 — 每步执行后 extract_step_experience()
  从execution_log的每条记录中提取:
  - drift_level/score → drift_warning
  - correction.action → rollback_lesson / reset_lesson  
  - comparison.consistent → planner_disagreement
  - 无异常 → success_pattern

Step 2: 总结 — 任务完成后 summarize_task()
  聚合所有步骤的经验, 生成:
  - 成功模式数/警告数/教训数
  - 整体评分(0.0-1.0, 基于权重计算)
  - 注入到语义/关键词/时间线三个通道
  
Step 3: 复用 — 新任务启动 retrieve_relevant_experiences()
  三路检索:
  - SQLite FTS5(内容LIKE匹配)
  - 语义通道(n-gram余弦)
  - 关键词通道(BM25)
  结果合并去重, 按评分排序
```

### 经验类型

| type | 含义 | success_rating |
|------|------|----------------|
| success_pattern | 顺利执行的步骤 | 0.9 |
| drift_warning | 发生目标漂移 | max(0.1, 1-score) |
| rollback_lesson | 需要回退 | 0.3 |
| reset_lesson | 完全重置 | 0.1 |
| planner_disagreement | 双规划器分歧 | 0.5 |

### 已知限制

1. **经验具体内容依赖execution_log** — 如果execute_plan的while循环提前退出(如experience_engine自身异常被except吃掉了), execution_log为空, 总结只有"评分=0.50, 模式=0个, 警告=0个, 教训=0个"。**必须确保experience_engine的import和init不抛出异常**。

2. **DriftLevel枚举兼容** — execute_plan中的step_log["drift"]["level"]是DriftLevel枚举对象, 注入到experience_engine时需要通过`hasattr(drift_level, 'value')`检测并转为字符串。

3. **同义词映射不覆盖所有领域** — 当前只有15个领域(JWT/数据库/用户/修复/重构/代码/系统/测试/性能/安全/接口/设计/配置/文档)。新领域需要手动添加。

4. **跨任务检索精度** — 依赖n-gram hash(无sentence-transformers), 语义相近但词汇不同的任务可能匹配不到(如"JWT" vs "JSON Web Token")。
