## 关键教训

### 1. Path.home() 在cron环境不可靠
症状：`Errno 2: '1/.hermes/reports/...'`  
原因：cron运行时 `HOME` 环境变量未正确设置，`Path.home()` 返回 `/`  
修复：111个脚本全部改为 `Path("/home/administrator/.hermes")`

### 2. 文件注入导致全系统瘫痪
症状：9个Python脚本同时损坏，全部 `IndentationError`  
原因：execute_code用heredoc重写文件时，破坏了原始代码格式  
修复：从pyc备份剥离行号前缀恢复（`re.sub(r'^\s*\d+\|', '', content)`）  
**教训：** 修改生产文件必须用 `patch` 工具（精确字符串替换），不能用 heredoc 全量重写

### 3. 正则文件名匹配在中文环境下不可靠
症状：glob("*八_七条永久执行规则*.md") 有时匹配有时不匹配  
原因：Python UTF-8 locale在cron下可能未设置，中文文件名glob行为不确定  
修复：三层回退策略（glob→精确映射→stem扫描）

### 4. except: pass 是最危险的代码模式
统计：23处bare except，覆盖全部9个脚本  
风险：JSON解析失败、文件损坏、网络错误全部静默吞掉  
修复：每处改为精确捕获+日志

## 全网调研结论

### 上下文压缩/索引领域
- Claude Code: 用LLM压缩摘要→有信息损耗
- Mem0: 向量库语义检索→成本高
- LangGraph: 状态图→复杂度过高
- **Hermes索引-复原方案在个人Agent级别全球领先**

### 主动反馈领域
- AutoGPT/BabyAGI: 无推送，只有终端REPL
- TaskWeaver: 日志级别通知，无实时推送
- **Hermes的定时+子阶段双推送是最成熟的个人Agent实践**

### 中断恢复领域
- LangGraph: 检查点版本化（可借鉴加版本号）
- Saga模式: 补偿事务（Hermes不需要，单Agent无分布式问题）
- **Hermes齿轮链式互验是独特设计**
