# v8.1 Pipeline 变更日志 — 2026-05-29

## 修复的P0问题

### 1. 多人物混合（0.59% = 588条/10万）

**症状**: 一条prompt中出现"一名女性和一名男性"（2个不同性别的人），但A01只定义了1个人物。

**根因**: `pick()`从维度库取条目时，维度库中A02/B01/C02/D01等维度总共298条含"和一名""两位""一对"等多人物词。A01选了"少女"，但A02却取到"一名留着黑色长发的年轻亚裔女性"。

**修复**: 
- `pick()`增加MULTI_WORDS过滤：`{'和一名','和一位','两名','两位','三名','三位','一对','二人','两人','她们','他们','她和','他和','一男一女'}`
- 维度库v3全库已多人物过滤（原位更新）

### 2. 性别断裂（v8旧版60%→修复后0%）

**症状**: "年轻男子"+"蕾丝文胸"、"中年男子"+"金色长直发"

**根因**: 
- gender_filter是软过滤（只过滤含"她/女性"代词的条目）
- D01_服装款式大量条目（"蕾丝文胸""比基尼"）不含代词未被过滤
- B01场景条目"女性站在浴室"含有性别矛盾

**修复**:
- `hard_filter_gender()`: male时正则排除含FEMALE_CLOTH词的条目
- B01性别匹配: B01条目按A01性别过滤后才取
- 最终代词一致性检查: txt中"她"和"他"的计数与A01性别比较

### 3. 超长句子（>1200字符）

**症状**: 风格种子(300字)+各维度补字数+逗号拼接 → 800+字长句

**修复**: 后置门增加上限 `len(txt) > 1200`，超过直接拒绝

### 4. 线程写入行级交叉

**症状**: 不同线程同时write到同一个文件 → 一行内包含多个prompt的碎片

**修复**: 每个线程先写到独立的缓冲区(thread_buffers)，每满100条batch flush一次到文件。仅持有write_lock时写文件，其他线程在此期间不写。
