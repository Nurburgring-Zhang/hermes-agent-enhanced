# PromptLibraryNode 总纲多维度格式规范（2026-05-27）

## 背景
用户反馈所有模板输出的总纲部分太干，只有一行参数罗列（"风格：XX | 色调：XX"）。要求多维展开。

## 通用规则

所有非设计类模式（故事板9种+绘本+短剧+儿童4种）的总纲必须包含3-4个维度的小标题段落，
每个维度3-5句话。`---` 分隔总纲与具体内容。

## 各模式总纲格式

### 故事板类（电影分镜/广告故事板/动画故事板/漫画分镜/MV故事板/教程步骤/短视频分镜/品牌故事板/剧情分镜）

```
# 【{mode}总纲】
## 一、故事基调
整体风格为{style}，色彩调性偏向{color_tone}。景别以{preferred_shot}为主，运镜采用{camera_style}的方式。

## 二、角色与场景
角色设定：{character_desc}。场景背景：{env_desc}。

## 三、叙事结构
共{shot_count}个镜头。每个镜头围绕主题展开，镜头之间要有因果推进关系。

## 四、画面风格指引
画面要有{mode}特有的叙事语言。禁止使用抽象情绪词，用具体可见的动作和场景来传递情感。
---
```

### 绘本模式

```
# 【绘本总纲】
## 一、故事主题
{topic}。风格偏向{style}，整体色调为{color_tone}。适合{age_group}阅读。

## 二、画面与文字节奏
共{pages}页，文字量{text_amount}。画面叙事和文字叙事交替推进。

## 三、创作方向
用孩子的视角看世界。不说教，让故事自己说话。
---
```

### 短剧模式

```
# 【短剧总纲】
## 一、故事设定
{topic}。风格为{style}，节奏{rhythm}，色彩调性偏向{color_tone}。

## 二、镜头与节奏
共{shot_count}个镜头，竖屏9:16。运镜以{camera_style}为主。

## 三、画面风格指引
每个镜头要有画面感，禁止抽象词。
---
```

### 儿童内容

```
# 【{mode}总纲】
## 一、故事主题
{topic}。画风采用{art_style}，适合{age_group}年龄段。

## 二、内容结构
共{count}个片段/页。先引起好奇，中间有波折，最后开心收尾。

## 三、创作方向
用孩子的语言讲故事，不说教。画面活泼有趣，色彩鲜明。
---
```

## 设计模式例外
设计模式输出的是AI提示词（给SD/Flux等工具），不需要叙事类总纲。已经在_build_*_prompt中有角色设定+设计哲学+输出要求三段式结构。
