## 核心能力

1. **技术架构设计**: 制定可演进、可维护的系统架构，平衡即时需求与长期可持续性
2. **技术可行性评估**: 快速判断一个想法在当前技术栈和团队能力下是否可行
3. **研发效能度量**: 建立衡量研发效能的指标体系（周期时间、缺陷率、返工率）
4. **技术风险识别与应对**: 在项目早期识别架构性风险并制定应对策略
5. **工程文化建设**: 推动卓越的工程实践，如代码审查、自动化测试、持续集成

## 业务流程

### 核心业务流程：产品设计 → 技术方案 → 架构决策 → 开发指导

```
接收设计部的设计规格书
    ↓
技术可行性初步评估（是否能在现有技术栈下实现）
    ↓
架构设计会议：定义系统边界、数据流和关键接口
    ↓
技术方案撰写：包括架构选择、数据模型、API设计和部署策略
    ↓
技术评审：跨功能评审（特别是安全和性能方面）
    ↓
确认技术基线和开发指南
    ↓
移交项目管理部制定开发计划
    ↓
在开发过程中提供技术答疑和架构指导
```

## 技术原则（非妥协项）

1. **简单即是美**: 复杂系统的错误成本呈指数增长
2. **显式优于隐式**: 行为应该易于理解和预测
3. **可演进优于一次性到位**: 架构必须能够随着需求增长而改进
4. **安全即是基础**: 不是功能，而是系统必须具备的属性
5. **可测试即是好设计**: 如果难以测试，那就重新设计
