# 纯本地提示词组合架构（2026-05-26 格林主人指出）

## 核心发现

API组合的问题：**55%的错误来自API自己编造**，而不是维度库或组合规则的问题。
格林主人指出：维度库打好标了、规则完备了，为什么还要API来组合？

## 架构方案

### 输入
- `01_模板结构.txt` — 九层结构定义
- `13_场景库.txt` — 127个场景片段（10大类：晨间/通勤/工作/午后/黄昏/深夜/旅行/情绪/写真/社交）
- `维度库_api_v7_governed/` — 17维度治理后库（143万条标签化数据）
- 14个风格库文件

### 组合算法

```
1. 从13_场景库.txt随机选1场景（127个，按比例70:30）
2. 从14风格库随机选1风格（C01）
3. 从维度库取:C02光照/C03色彩/C04构图/D03材质/D06氛围/D05天气各1
4. 按九层结构填充:风格名→镜头构图→场景主体→材质细节→层次→色彩→光影→氛围
5. 90条规则+矛盾检测过滤
6. 输出完整prompt
```

### 不需要API的理由

| 维度 | API组合 | 本地组合 |
|------|---------|---------|
| 生成速度 | 5-8秒/条 | 毫秒级 |
| 成本 | $300/10万条 | 0 |
| 新增元素 | 55%乱编率 | 0% |
| 一致性 | 每次随机 | 完全可复现 |
| 控制力 | 弱（AI不听话） | 强（规则完全可控） |
