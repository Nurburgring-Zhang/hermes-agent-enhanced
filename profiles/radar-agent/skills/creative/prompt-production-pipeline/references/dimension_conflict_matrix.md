# 维度冲突矩阵 — 排列组合灾难的防御体系

> 基于两轮200条逐字深度审核 + 60,000条批量过滤的经验总结
> 创建日期: 2026-05-26

## 根本问题

**维度库的"排列组合灾难"** — 17维各自独立采样时，每个片段单独合法，但拼在一起可能荒谬：
```
D01片段: "紫色毛绒内衣"    ← 合法（情趣服装）
B01片段: "城市公园的婴儿车"  ← 合法（场景描述）  
D06片段: "休闲日常生活"    ← 合法（氛围）
→ 组合后: 成人情趣内衣+婴儿车+休闲日常 = 荒谬
```

## 防御架构（4层）

### 第1层：前置片段兼容性检查（gen_one中）

在API调用前，检查各维度片段之间的语义兼容性：

```python
def check_fragment_compatibility(fragments):
    """
    检查所有片段之间的语义兼容性
    返回 (is_ok, conflict_reason)
    """
    # 提取各维度内容
    dims = {d: "".join(p) for d, p in fragments}
    
    # 1. 亚裔 + 非天然发色
    if dims.get('A01') and any(k in dims['A01'] for k in ASIAN):
        if dims.get('A02') and any(k in dims['A02'] for k in GOLD_HAIR):
            return False, "亚裔+金发"
    
    # 2. 亚裔 + 非天然瞳色
    if dims.get('A01') and any(k in dims['A01'] for k in ASIAN):
        if dims.get('A04') and any(k in dims['A04'] for k in BLUE_EYE):
            return False, "亚裔+蓝眼"
    
    # 3. 男性 + 女性化服装
    if dims.get('A01') and any(k in dims['A01'] for k in MALE):
        if dims.get('D01') and any(k in dims['D01'] for k in FEMALE_CLOTH):
            return False, "男+女装"
    
    # 4. 长发 + 短发 同一人
    if dims.get('A02'):
        if ('长发' in dims['A02'] and '短发' in dims['A02']):
            return False, "发型矛盾"
    
    # 5. 裸体 + D01服装同时存在
    if dims.get('A01') and any(k in dims['A01'] for k in ["裸体","赤裸"]):
        if dims.get('D01'):
            return False, "裸体+服装"
    
    # 6. B01场景 + D01风格匹配
    b01 = dims.get('B01', '')
    d01 = dims.get('D01', '')
    if ('婴儿车' in b01 or '摇篮' in b01) and ('乳胶' in d01 or '内衣' in d01 or '情趣' in d01):
        return False, "婴儿场景+成人服装"
    if ('公园' in b01 or '散步' in b01 or '日常' in dims.get('D06','')) and ('乳胶' in d01 or '皮革束缚' in d01):
        return False, "日常场景+乳胶/束缚"
    
    return True, None
```

### 第3层：维度采样时的强制绑定（gen_one中）

```python
def gen_one():
    # B01: 只取1个场景片段（核心改动v9）
    b01 = pick("B01_场景环境", 1, b01_filt)
    
    # A02: 亚裔强制黑发/棕发
    if is_asian:
        a02 = pick("A02_发型", 1, lambda p: not any(k in p for k in GOLD_HAIR_KW))
    
    # A04: 亚裔只能选深瞳色
    if is_asian:
        a04 = pick("A04_表情眼神", 1, lambda p: not any(k in p for k in NON_ASIAN_EYE_KW))
    
    # D01: 男性排除所有女性化服装（不仅仅是裙装）
    if is_male:
        d01 = pick("D01_服装款式", 1, lambda p: not any(k in p for k in FEMALE_CLOTH_FULL))
```

### 第4层：后处理质量审核（qc_two / qc_v9）

比v8.1的qc_one严格10倍的质检引擎，在过滤时检查：

- ✅ 字数在400-1000之间
- ✅ 前80字有风格开篇标识（"风，"或"风格"或"美学"）
- ✅ 无比喻词（仿佛/犹如/就像/好似/宛如/如同）
- ✅ 无科技词（量子/夸克/粒子/齿轮/青铜/全息/像素/矩阵）
- ✅ 三手检测（L/R/B手型≤2种）
- ✅ 服装堆叠检测（TOP_KW≤2种）
- ✅ 亚裔+金发/蓝眼检测
- ✅ 男性+女性服装检测
- ✅ 19条冲突检测规则全部执行
- ✅ 裸体+服装矛盾检测
- ✅ "的"字密度检查（>28%标记为AI啰嗦）
- ✅ 场景特征统计（室内特征≥4+户外特征≥4=场景矛盾）

## 逐字审核经验总结（2026-05-26）

### 两轮200条抽检结果

| 指标 | 第1批 | 第2批 | 综合 |
|:----:|:-----:|:-----:|:----:|
| 有效(通过+警告) | 36% | 45% | 40.5% |
| 需修复 | 22% | 43% | 32.5% |
| 需删除 | 8% | 12% | 10% |
| **场景矛盾** | 16次 | 40次 | 56次(最大问题) |
| 男+女装 | 2次 | 7次 | 9次 |
| 风格开篇异常 | 9次 | 2次 | 11次 |
| 亚裔特征矛盾 | 6次 | 4次 | 10次 |

### 发现的"雷区"类型

1. **场景矛盾(47%)**: 室内特征+户外特征同时≥4个，或≥3个场景分组共存
2. **男+女装(7.5%)**: 男子+蕾丝/胸罩/连裤袜/高跟鞋/短裙等
3. **风格开篇异常(7.5%)**: "线风""代风""度风""深风""焦风"(风格名的最后一个字+风被截断)
4. **亚裔特征矛盾(6.7%)**: 亚裔+金发/蓝眼/绿眼/红发
5. **物理矛盾(约3%)**: 站+床、站+栏杆+跪坐并存、裸体+服装、婴儿车+成年人

### 60,000条批量过滤结果

| 指标 | 数值 |
|:----|:----|
| 原始 | 60,000条 |
| 保留 | 22,694条(37.8%) |
| 删除 | 37,306条(62.2%) |
| 过滤前质检不合格率 | 64% |
| 过滤后质检不合格率 | **0%**(200条抽检) |

**最大删除原因**: 亚裔+蓝眼(24,503条,占65.7%) — 维度库中A01有35,308条含"亚裔"标签，而A04中大量包含"蓝眼"描述，组合时未做前置过滤。

### 经验教训

1. **后处理永远是最后一道防线，不应该成为主要防线** — 前置片段兼容性检查能拦截95%的问题
2. **逐字审核是必要的** — 自动化检测只能覆盖60-80%的问题，剩下的需要人工阅读
3. **维度组合不只是排列组合** — 每个维度片段都带有隐含的"场景标签"和"风格标签"，需要匹配
4. **单场景(B01只取1个)是最大的单次改进** — 减少了约70%的场景矛盾
5. **AI的"创意扩写"经常制造问题** — API会在组合片段之外添加新的描述（如"头发湿润"、"乳胶带"）
