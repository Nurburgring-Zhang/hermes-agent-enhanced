# 极端严苛商用级代码审核方法论

## 触发条件

当格林主人要求 "深度代码审核"、"代码审计"、"极端严苛审核"、"商用级审核"、"逐行审核" 时加载此skill。

**扩展触发**：当要求"全面研究+审核+优化方案"时，加载本skill + `references/racing-game-audit-methodology.md`（全频谱审核方法论）。

**竞速游戏审核**：当目标项目是竞速/赛车/3D驾驶游戏时，同时加载 `references/racing-game-physics-benchmarks.md` 作为参数基准对照表。

## 核心流程

### 阶段1：三路并行深度审核（并行启动3个delegate_task）

对N个核心文件（通常3个：建库脚本+组合引擎+质量过滤器），每个文件分配独立的leaf子Agent进行极端严苛审核。

每个leaf子Agent的审核清单：

```
1. ✅ 功能完整性 — 所有功能真实实现？有无占位符(TODO/FIXME/pass/.../NotImplementedError)？
2. ✅ 代码安全 — 文件I/O在try内？无裸except？无路径穿越？无SQL注入？无硬编码密码？
3. ✅ 类型注解 — 所有函数有完整类型注解？
4. ✅ 错误处理 — try/except精准捕获异常类型？
5. ✅ 性能 — O(n²)？一次性大文件读取？可优化点？
6. ✅ 逻辑正确性 — 是否符合需求文档/规则？
7. ✅ 输出格式 — 是否符合规范？
```

### 阶段2：汇总缺陷报告

从3个子Agent收集结果，汇总为统一缺陷表，按优先级排列：
- P0（致命）：语法错误、数据丢失风险、安全漏洞
- P1（高）：功能缺失、类型注解完全缺失、过滤逻辑误杀
- P2（中/低）：代码品质、性能优化

### 阶段3：并行修复（并行启动3个delegate_task）

每个leaf子Agent负责修复对应文件的所有缺陷，按优先级顺序修复。

修复原则：
- 每次只改一行或一个函数块（使用patch操作，不是全文件重写）
- 每个修复后验证语法正确性
- 不改变文件的其他逻辑

### 阶段4：回归验证

对修复后的每个文件运行语法检查 + 功能测试，确认修复未引入新问题。

### 格林主人永久禁令（审核时必须检查）

审核时必须逐行搜索以下模式：

| 检查项 | 检测规则 | 严重度 |
|--------|----------|--------|
| 占位符 | TODO / FIXME / pass / raise NotImplementedError / ...（三个点） | P0 |
| 裸except | `except:` 单独出现（会捕获SystemExit/KeyboardInterrupt） | P1 |
| SQL注入 | `execute(f"{...}")` 而非参数化查询 | P0 |
| 硬编码密码 | `password="xxx"` 非环境变量 | P1 |
| 文件I/O无保护 | open()不在try内 | P0 |
| subprocess无超时 | run()无timeout参数 | P1 |
| 函数过长 | >100行 | P2 |
| 嵌套循环 | for内for可能O(n²) | P2 |
| 无类型注解 | def foo(x) 无 -> type | P1 |

### 常见审核发现模式

1. **过滤逻辑误杀（最常见P1缺陷）**：
   - 单字过滤词（"若"在"若有所思"、"胸"在"心胸"）→ 必须用词边界或上下文
   - 科技词过于宽泛（"霓虹"、"机械"在人物描述中合法）→ 缩小或加上下文
   - 字符串拼接漏逗号（`"床上" "沙发"` → Python隐式拼接为"床上沙发"）→ 语法错误

2. **三手检测陷阱**：
   - `"指"` 在动词列表 → 误杀"右手指尖"、"指尖轻触"
   - 统计次数不等于矛盾检测 → 应当检测逻辑矛盾而非累计次数

3. **场景矛盾根因**：
   - 维度库条目本身就含场景词（C02光照+户外/室内混合）→ 需要源数据清洗
   - 组合引擎无法完全解决 → 需要在建库层做场景分裂

### Pitfalls

- ⚠️ 不要一次性全文件重写 — 用patch逐行修复，每个修复独立验证
- ⚠️ 审核子Agent可能超时（大文件）— 设置足够timeout（120s+）
- ⚠️ 修复子Agent可能超时（太多patch需要逐个验证）— 设置足够timeout（300s+）
- ⚠️ 注意子Agent可能修改了父Agent已读取的文件 — 修复前重新读取
- ⚠️ 三路审核可能发现重叠缺陷 — 汇总时去重
- ⚠️ 格林主人的比喻词禁令包括单字"若" — 已从禁止列表中移除（误杀过高），但须确认格林主人是否接受

## 回滚方案
### 代码回退
1. `git revert HEAD` 撤销最后一次提交
2. `git stash` 恢复工作区状态
3. 重新运行测试套件确认无回归

### 紧急回滚
1. `git reset --hard HEAD~1` 硬回退
2. `git push --force-with-lease` 推送
3. 通知团队变更已回退
