# Hermes 6 Essential Skills

## 核心理念

基于龙虾牧羊人精挑细选的6个Hermes必备Skills，全部融入Hermes底层能力。

**核心主张**：装上这6个，你的Hermes才算真正能打。剩下的按需添加。

## Skill 1: 对抗式内容审核（三Agent互审）

### 原则

发布前就知道哪里要改，而不是发布后才知道踩坑了。

### 机制

```python
# 三个Agent互怼：
笔杆子Agent: 负责写初稿
参谋Agent:  专门挑毛病——五维批判（标题/结构/数据/价值/篇幅）
裁判Agent:  打分，>=9分才放行
```

### 触发条件

- 所有内容生产任务（文章/汇报/方案/代码/设计等）
- 特别是要发出去给很多人看的那种

### 与Hermes现有能力的融合

```yaml
integration:
  - 笔杆子Agent → delegate_task作为Content Writer Worker
  - 参谋Agent → production-reliability-engine的CriticAgent审查能力
  - 裁判Agent → agent_committee专家委员会打分
  - 评分<9 → 自动进入达尔文互优化流程（darwin-skill-evolver-bridge）
  - 三重评审 → cognitive-distillation-ppt-director的评审三检查

activation:
  trigger: "任何需要外部发布的内容输出"
  mode: "自动执行，在write_file后自动触发"
```

### 三Agent具体指令

#### 笔杆子Agent指令
写初稿，专注于内容创作本身，不考虑审查。

#### 参谋Agent指令（五维批判）
```yaml
critique_dimensions:
  - 标题: "标题是否吸引人？是否有价值主张？"
  - 结构: "逻辑结构是否清晰？段落衔接是否自然？"
  - 数据: "数据是否准确？来源是否标注？"
  - 价值: "读者读完能获得什么？"
  - 篇幅: "内容是否冗余？能否精简？"
```

#### 裁判Agent指令
对参谋的批判打分（1-10）：
- >= 9分：放行
- < 9分：返回给笔杆子修改

### 迭代循环
每次修改后重新触发，直到>=9分或达到最大迭代次数（3轮）。

## Skill 2: AI文本润色

### 原则

去掉AI味，让文章像人写的。不是改错，是"去AI化"。

### 润色技巧

```yaml
techniques:
  - 打破句式对称: "别全是'首先、其次、最后'"
  - 增加口语化: "说实话、没想到、绝了"
  - 增强情绪感染: "反问句、感叹词"
  - 具象化表达: "'效果很好'→'转化率从3%飙到12%'"
  - 去掉过渡词: "值得注意的是/不可否认的是/值得一提的是"
  - 缩短段落: "超过4行就拆"
```

### 触发条件

- 所有AI生成内容的最后一步
- 用户说"帮我润色"
- 用户说"去掉AI味"

### 与Hermes现有能力的融合

- 已注入 `hermes_retrospect.py` 的质量评估维度
- 写入user profile偏好（格林主人禁止符号标记的格式要求）
- 每次内容输出后自动做"去AI化"检查

### 进阶：文学大师规范（来自SOUL.md九面人格架构）

当润色任务要求高质量中文写作时，激活以下文学大师专属规范：

#### 奥威尔五条规则（强制）
1. 拒绝陈词滥调——不用"如火如荼""史无前例"等空泛词汇
2. 恰当使用短句——用"开始"不用"着手进行"
3. 主动语态为主（>80%句子）
4. 非必要避免外来术语
5. 宁可打破规则，也不写荒谬的句子

#### 禁用词表
  
| AI痕迹表达 | 自然替换 |
|------------|---------|
| 呈现/展现/彰显 | 直接说"是什么" |
| 旨在/致力于/着力于 | 用"想/要/在做" |
| **赋能/助力/加持**（最恶劣黑话） | 用"帮/让/靠" |
| 显著/大幅/明显 | 用具体数字 |
| 进行了XX/具备XX能力 | 直接说动词或"能/会" |

#### 句式禁区与强制规范

禁止出现：
- 「随着...的发展/深化」开头
- 「在...的背景下」「对于...而言」（机械连接词，直接删除）
- 连续超过2个「的」字连用
- 一句话超过50个字
- 「不仅...而且...」虚假范围

#### SCQA金字塔结构（文章/报告类强制适用）
```
标题：包含核心利益点
开头：S(情景)→C(冲突)→Q(疑问)→A(核心假设，直接亮出观点)
正文：小标题+核心句+详细阐述+关键结论单独成段
结尾：最多3句话，无「综上所述」
```

#### 50分制质量自评
| 维度 | /10 | 检查 |
|------|-----|------|
| 直接性 | /10 | 直截了当还是绕圈铺垫？ |
| 节奏感 | /10 | 长短句交错还是机械重复？ |
| 信任度 | /10 | 尊重读者还是过度解释？ |
| 真实性 | /10 | 像真人说话还是机器生成？ |
| 精炼度 | /10 | 还有可删减内容吗？第一稿后通常删除50% |

## Skill 3: 素材搜索大师

### 原则

10分钟搞定素材，不花2小时找东西。

### 素材速查表

| 类型 | 推荐平台 |

| 图库 | Unsplash, Pexels, Pixabay |
| 视频 | Pexels Video, Coverr |

```python
SEARCH_FORMULA = "{行业} + {成功/失败} + {案例/数据} + {年份}"
# 例: "AI 成功案例 2024"
```

```yaml
monitored_platforms:

  - 抖音热点
  - 知乎热榜
  - 百度热搜
  - 小红书热榜

  - 突发不可预测热点: "突发事件/爆款内容"

TREND_FORMULA = "{热点事件} + {自身领域} + {独特视角} = 爆款选题"
# 例: 热点"董明珠称不用女性员工"
#     选题"董明珠这句话，暴露了职场最深的偏见"
```

### 与Hermes现有能力的融合

## Skill 5: 多Agent调度中心

### 原则

丢个需求下去，坐等结果。不要手忙脚乱任务堆积。

### 调度流程

```
任务输入 → 任务解析 → 任务拆解 → Agent分配
    ↓                              ↓
结果汇总 ← 进度监控 ← 执行中 ← Agent1/2/3并行
```

### 适用场景

- 复杂项目需要多轮协作
- 需要同时处理内容、素材、发布多个任务
- 日更多条内容需要流水线生产

### 与Hermes现有能力的融合

```yaml
integration:
  - 任务解析 → goal-hive-orchestrator的Master拆解
  - Agent分配 → delegate_task并行worker
  - 进度监控 → BBS任务账本（hive_bbs.json）
  - 结果汇总 → Master验收
```

## Skill 6: 记忆与知识库

### 原则

让你的AI越来越懂你。每次对话AI都像陌生人 = 失败的记忆系统。

### 知识库结构

```
knowledge_base/
├── 个人风格/
│   ├── 写作风格（用词偏好、句式习惯）
│   ├── 常用句式
│   └── 禁忌词
├── 行业知识/
│   ├── 行业词库
│   ├── 术语表
│   └── 趋势
├── 爆款案例/
│   ├── 标题库
│   ├── 结构库
│   └── 金句库
├── 内容规则/
│   ├── 平台规则
│   ├── 敏感词
│   └── 合规
└── 模板库/
    ├── 标题模板
    └── 文章模板
```

### 重要原则

| 做什么（存到这里） | 不做什么 | 存储位置 |
|--------|---------|---------|
| 用户偏好/习惯/禁忌词 | 任务进度/会话结果 | **memory → user** |
| 系统环境事实/工具配置/项目约定 | 临时TODO状态 | **memory → memory** |
| 已验证的工作流/方法论/技术模式 | 今日已完成的事项 | **skill_manage** |
| 结构化领域知识/多轮对话累积的事实 | 过7天就没用的信息 | **fact_store** |
| 情感记忆/用户日常压力源/内部梗 | 一次性会话日志 | **Holographic Memory** |
| 历史会话检索 | 不存（用搜索代替） | **session_search** |

### 分片存储规则（关键！）

系统有三种记忆存储，**各司其职、不混用**：

| 存储类型 | 存放什么 | 典型大小 | 何时用 |
|---------|---------|---------|-------|
| **memory**(memory工具) | 用户画像 + 环境事实 + 操作教训 | ~2200 chars | 每次对话自动注入 |
| **fact_store**(fact_store工具) | 结构化知识 + 多轮累积事实 | 无限 | 任务需要特定知识时主动查询 |
| **skill_manage**(skill_view工具) | 可复用的工作流 + SOP | 无限 | 遇到匹配任务时自动加载 |

**不要去memory里塞任务进度和临时状态**。这会让记忆系统变脏、信任评分降低。
**不要用memory存储结构化知识**——fact_store专门为此设计，带分类和标签检索。
**不要把方法论写到memory**——存成skill，下次自动加载。

### 自动保存规则

```yaml
auto_save_rules:
  - 格林主人每次纠正 → 存到memory（user profile）
  - 任务特定发现（可复用的工作流） → 存为skill
  - 重复3次以上的模式 → 自动创建skill候选
  - 超过1小时的对话 → 关键事实存到fact_store
  - 复杂任务执行完 → 复盘要点存到memory（memory notes）
  - 会话结束 → 新发现的用户偏好存到memory（user profile）
```

### 与Hermes现有能力的融合

```yaml
integration:
  - memory工具 → 用户偏好和事实（user profile）
  - memory工具 → 环境事实和约定（memory notes）
  - Holographic Memory → FTS5+信任评分的持久记忆
  - Hy-Memory L1-L3 → 事实提取/场景归纳/画像生成
  - session_search → 会话检索
  - cross_session_cache → 跨会话缓存
  - skill_manage → 可复用的工作流
  - fact_store → 深度结构化记忆

auto_save_rules:
  - 格林主人每次纠正 → 自动保存到memory
  - 任务特定发现 → 保存为skill
  - 重复3次以上模式 → 自动创建skill候选
  - 会话结束 → 自动注入关键信息到memory
```

## 扩展用法: 外部知识注入的质量控制（2026-06-06）

hermes-6-essential-skills 可以作为**外部知识集成质量控制层**使用 — 在从文章/论文/开源项目提取方法论并注入Hermes时，使用对抗式内容审核检查注入质量：

```python
# 外部方法论注入的质量门
Step 1: 提取方法论
Step 2: 笔杆子Agent写注入方案
Step 3: 参谋Agent五维批判（完整性/一致性/可行性/无冲突/覆盖度）
Step 4: 裁判Agent打分（≥9分放行）
Step 5: 执行注入（创建Skill/脚本/SOUL.md规则）
Step 6: 验证注入是否达到预期效果
```

素材搜索大师(Skill3)和热点雷达(Skill4)同时用于**发现新的外部方法论**——采集系统发现高价值文章时，自动触发注入流程。

## 陷阱

- **互审不是无限循环**：设定最大迭代次数（默认3轮），超过后由格林主人决断
- **润色不是改错**：去AI味不等于把内容改成人话就完事——要保留原文的信息量和专业性
- **热点检测延迟**：采集周期是3小时，所以热点检测依赖采集频率。如果热点在分钟级变化的场景，需要单独加高频采集
- **调度有成本**：不是所有任务都值得拆。5分钟能搞定的简单问答不要用调度中心
- **记忆不是垃圾桶**：不要往记忆里塞任务进度和临时状态。这会让记忆系统变脏、信任评分降低
