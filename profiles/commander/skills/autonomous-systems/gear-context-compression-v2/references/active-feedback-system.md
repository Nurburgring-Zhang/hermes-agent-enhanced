# 主动状态反馈系统参考

## 组件清单

| 脚本 | 路径 | 频率 | 职责 |
|:-----|:-----|:------|:-----|
| status_reporter.py | scripts/status_reporter.py | 每40分 + 每2小时force | 无对话时自动推送系统状态 |
| feedback_push.py | scripts/feedback_push.py | 按需（齿轮驱动） | 任务子阶段完成时推进度 |
| session_init_check.py | scripts/session_init_check.py | 对话启动 | 启动时检测异常并推送告警 |

## 推送内容

### status_reporter 推送格式

```
🤖 Hermes 系统状态 · 05-27 14:16
✅ 齿轮系统: HEALTHY
⏱ 持续运行: 0天2小时

📡 采集: 27975条raw → 27600条clean
   最新: 2026-05-27 14:16 | 未评分: 4252
📤 推送: 今日108条 | 累计3817条
   最新: 2026-05-27T13:01
🔄 自增强循环: 第1779862507轮

---
💬 有任务直接说，我在线
```

### feedback_push 格式

```
⚙️ **阶段名** #task_id...
✅ 当前做完了什么
→ 下一步: 计划做什么
```

## 格林主人要求

1. **40分钟无对话 → 自动推送** — 防止用户以为系统停了
2. **2小时强制推送** — 确保即使有对话也有定期状态
3. **任务子阶段完成 → 推送** — 长任务不沉默
4. **对话中断恢复后 → 推送** — 通知用户接上了
5. **启动检测发现异常 → 告警** — 不等用户问

## PushPlus 通道

### Token获取（三级回退）
1. config.yaml → pushplus.token
2. .env → PUSHPLUS_TOKEN=(支持export前缀和引号包裹)
3. 返回空字符串

### 调用限制
- 每日未限制但建议≤200条/天（免费版）
- 单条消息≤20000字节（wx html模式下约10000中文字）
- timeout=15秒

### 已知问题
- 偶尔返回"服务端验证错误" — 重试1次
- markdown模板在微信中可能渲染不一致（emoji+粗体组合）
