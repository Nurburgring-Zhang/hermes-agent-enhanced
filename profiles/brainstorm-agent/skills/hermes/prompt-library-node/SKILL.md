## 总纲格式（4个总纲统一 — 故事板/绘本/短剧/儿童）

```
【电影分镜总纲】
1、整体视觉风格：
    整体风格为电影感，色彩基调偏向暖色调。
2、角色物品设定：
    角色描述内容（多角色用换行分隔）
3、道具/武器：
    待补充。
4、场景设定：
    场景描述内容
5、氛围与画质标准：
    风格叙事描述
6、声音设定：
    根据场景氛围搭配环境音效和配乐。
7、核心叙事设定：
    共N个镜头/页，围绕主题展开。
```

**总纲硬规则（格林主人反复强调）：**
- 编号必须是 `1、2、3、`（中文逗号+整数+顿号）
- 字段名用中文文字加顿号：`1、整体视觉风格：`
- 子内容统一缩进，`角色物品设定`和`场景设定`中的字段描述直接续写
- 总纲内**禁止空行** — 所有行紧凑排列
- 总纲与正文之间用一个空行分隔

## 🚨 角色特征 vs 画面 的维度区分陷阱（2026-06-01 新增）

**这是LLM最容易搞混的一个维度，格林主人已经因此发火多次。**

### 核心问题

LLM常常把"角色特征"写成"角色在这个镜头里的动作和状态"，跟`画面`维度完全重叠。

### 正确界定

| 维度 | 应该写什么 | 不应该写什么 |
|------|-----------|-------------|
| **角色特征** | 相貌/服装/身体形态的**实质性变化**（换衣服了、受伤了、变老了、新角色登场时的外貌描写）。无变化时整行消失。 | ❌ 角色在做什么动作（跑/跳/拉/坐）<br>❌ 角色当前的表情情绪<br>❌ 角色的当前位置<br>❌ 角色和环境的互动 |
| **画面** | 角色在做什么动作 + 场景里发生了什么 + 光线/色彩氛围。这是主要的叙事载体。 | ❌ 重复角色特征里的外貌描述 |

### 典型案例分析（格林主人2026-06-01纠正）

**错误输出：** Shot 2 角色特征写着"小兔子双手紧紧抱住萝卜的叶子，双脚蹬在泥土里，耳朵向后贴紧脑袋"
→ 这是动作叙事，属于`画面`维度，角色外貌/服装无变化，`角色特征`行应该消失。

**正确做法：**
- 角色特征：无变化 → 整行消失
- 画面：小兔子使出全身力气向后拉扯萝卜，泥土在它的脚掌下被蹬出浅浅的坑...

#### 第一类：format_templates / 绘本输出格式（5处相同）

### 三个核心改动点

1. **触发条件收紧：** `有角色特征变化时` → `仅在角色外貌/服装/状态有实质性变化时（换装/变脏/受伤/新增饰品等可见变化）` -- 给出具体例子，LLM更容易理解什么算"变化"
2. **增加负面约束：** 新增 `禁止写角色动作叙事（那是画面维度的事）` -- 明确告诉LLM这个行不是用来写动作的
3. **删除模糊词：** 删掉 `表情变化` -- 因为LLM会把"表情变化"理解成"描述当前表情"（那属于画面维度）

### 格林主人会检查的维度边界（他的检查逻辑）

他会把输出贴过来，直接指出"这一行写的不是角色特征，是画面叙事"。检查粒度到每个镜头的每一行。所以不要心存侥幸——如果他问"这是角色特征吗"，那肯定是有问题。

关键测试：提取角色特征行，看它们组成的故事是"角色身上发生了什么变化"还是"角色做了什么事"——前者正确，后者错误。

## 🔴 变化标注规则（V20.6最终版 — 格林主人最新指令 2026-06-01）

**覆盖范围：** 所有叙事型模板（故事板9种 + 绘本 + 短剧 + 儿童4种）

**核心规则（2026-06-01更新）：**
1. **只追踪两个维度：场景 和 角色特征。** 不再追踪服装/道具/时间变化。
2. **变化通过分镜场景/角色特征两个字段表达：** 不再使用独立的「变化标注：」行。
3. **有变化才出现：** 场景变化时输出「分镜场景」行（完整场景描述2-4句），角色特征变化时输出「角色特征」行（完整角色描述2-3句）。无变化时整行消失（包括字段标题本身）。
4. **两者都变化时各行一行。**
5. **维度名称用中文文字，不用emoji或特殊符号。**

**字段顺序（统一）：**
```
【Shot N】
景别：（始终输出）
分镜场景：（场景大变化时输出完整描述，无变化时整行不出现）
角色特征：（角色特征大变化时输出完整描述，无变化时整行不出现）
画面：（始终输出）
运镜：
转场：
时长：
叙事功能：（必要时才输出）
```

**正确与错误对比：**

| 情况 | 旧（错误） | 新（正确） |
|------|-----------|-----------|
| 无变化 | 分镜场景：无 + 变化标注：无 | （两行都不出现） |
| 有场景变化 | 分镜场景：菜园厨房（简短） | 分镜场景：中午·小兔子家厨房——灶台上铁锅冒着热气，窗外阳光透过木窗洒下暖金色光斑...（完整2-4句描述） |
| 角色变化 | 角色特征：小野猪出现 | 角色特征：小野猪穿着工装背带裤，圆滚滚肚皮上沾着面粉，鼻尖冒汗... |

**分镜场景/角色特征字段规则：**
- `景别`：始终输出，电影分镜标记了「（始终输出）」
- `分镜场景`：**仅在场景变化时输出**，无变化时整行消失
- `角色特征`：**仅在角色特征变化时输出**，无变化时整行消失
- `画面`：始终输出
- `叙事功能`：仅在必要时输出，大部分镜头不要

**分镜场景/角色特征/画面三者的顺序：**
```
【Shot N】
景别：（始终输出）
分镜场景：（仅在变化时输出）
角色特征：（仅在变化时输出）
画面：（始终输出）
运镜：
转场：
时长：
叙事功能：（必要时输出）
```

## 代码修改流程

1. **备份第一：** 改前必须备份到 `/mnt/d/Hermes/备份/`
2. **验证签名：** 备份恢复后必须 `grep -n 'def get_prompt|def IS_CHANGED' __init__.py` 确保是 `**kwargs`（备忘：`get_prompt(self, **kwargs)` 和 `IS_CHANGED(cls, **kwargs)`）
3. **统一修改：** 修一个模板必须修所有同类模板 — 变化标注规则要同步到故事板+绘本+短剧+儿童4种

### 对话带表情动作标注（儿童视频格式二独有）


**触发事件：** 我写了一版带表格、带`-`列表、带`|`分隔符、带专业术语的README，格林主人连续两次纠正："去掉所有无用符号"和"说人话的语气"。

### 核心规则：文档要像人写的，不是机器生成的

**1. 说人话**
- 不要用"本项目致力于""该节点旨在""提供了一种机制"这种机器人用语
- 用"它能干什么""怎么装""怎么用"这种大白话
- 直接描述痛点，不要绕弯子

**2. 去掉所有装饰符号**
- 不要 `---` 分隔线
- 不要 `#` 标题层级装饰（GitHub渲染自然会有层级，不要在md里过度使用）
- 不要 `**` 加粗
- 不要 `- ` 列表前缀
- 不要 `1. 2. 3.` 编号
- 不要 `|` 表格（除非数据密集、表格是最清晰的方式）
- 不要 `> ` 引用块
- 不要 `---` 水平线分隔章节

**3. 章节之间用空行分隔，不要用符号**

**4. 内容优先**
- 直接写功能，不要写"本README将介绍..."
- 安装步骤直接给命令，不要写"安装方法如下所示"
- 使用说明直接给操作，不要解释为什么这样操作

**5. 示例对比**

❌ 旧写法：
```
# PromptLibraryNode Pro V20.5

一款为 ComfyUI 设计的全能提示词创作与故事板生成节点。支持 25 种创作模式...

## 核心功能

- 读取本地提示词文件夹
- 支持 4 种读取模式
- 关键词/标签筛选
```

✅ 新写法：
```

## 它能干什么

**读提示词库**

```
