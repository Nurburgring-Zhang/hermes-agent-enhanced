## 一、角色特征维度定义（2026-06-02 精确化）

**关键规则：角色特征 ≠ 角色动作叙事**

LLM天然倾向在"角色特征"字段写"角色在做什么"——这是错误。角色特征是**外貌/服装/状态的变化记录**，不是动作描述。

```python
\"角色特征：仅在角色外貌/服装/状态有实质性变化时输出（换装/变脏/受伤/新增饰品等可见变化）。禁止写角色动作叙事（那是画面维度的事）。无变化时此行整行不出现。\"
```

**验证标准：**
- ✅ 小兔子换上了碎花围裙，白毛被蒸汽润湿 → 正确的角色特征（服装/状态变化）
- ❌ 小兔子抱住萝卜叶子使劲拔 → 这是动作叙事（应该放在"画面"维度）
- ✅ 场景不变、服装不变、外貌无变化 → 此行整行不出现

## 三、分镜头字段顺序（故事板模式）

场景或角色特征发生改变时：

```
【{mode}总纲】
1、整体视觉风格：
    整体风格为{style}，色彩基调偏向{color_tone}。
2、角色物品设定：
    角色描述（多角色每行一个，用换行区分）
3、道具/武器：
    待补充。
4、场景设定：
    场景描述（包括时间、地点、环境氛围）
5、氛围与画质标准：
    氛围描述
6、声音设定：
    声音描述
7、核心叙事设定：
    叙事设定
```

### 关键规则
- 编号从1到7，使用中文顿号（、）
- 总纲内禁止空行（格林主人硬性要求）
- 总纲内禁止使用任何符号：`#` `##` `**` `-` `- **`
- 缩进统一为4个空格，不使用变量拼接

## 二、分镜头字段顺序（故事板模式）

场景或角色特征发生改变时：

```
【Shot N】
景别：...（始终输出）
分镜场景：场景切换时输出详细场景描述，包括地点、时间、光线、环境细节（2-3句）（仅在场景变化时出现）
角色特征：（仅在角色特征变化时出现）
画面：...
运镜：...
转场：...
时长：...

```

### 关键规则
- 景别始终输出，放在第一行
- 分镜场景必须包含**完整的场景描述**（2-3句），不能只写地点名
- 分镜场景和角色特征在画面之前
- 无变化时整行不出现
- 叙事功能仅在必要时输出

## 三、变化标注规则（2026-05-31 第二轮修正）

### 核心变更
| 旧规则 | 新规则 |
|--------|--------|
| 无变化写「无」 | 无变化时整段不输出 |
| 五维度逐项列出，无论是否变化 | 只写有变化的维度 |
| 使用emoji（🔄👤👗📦☀️） | 用中文文字（场景变化/角色变化/服装变化/道具变化/时间变化） |
| 使用符号标记（🔴 变化标注） | 不使用任何标记符号 |

### 输出格式
有变化时，在画面描述段后单独一行输出：

```
变化标注：场景变化——从菜园切换到厨房；时间变化——从清晨推进到中午
```

### 规则
- 无变化时整段不输出，不写"变化标注"字样
- 有变化时只写有变化的维度，不写"无变化"的维度
- 维度名称用中文文字，不用emoji或特殊符号
- 多个变化用分号分隔
- 严禁使用 `#` `##` `**` `-` `- **` `🔴` 等符号

### 应用范围

## 四、system prompt中的符号禁令（2026-05-31 新增）

### 在system prompt中禁止使用的符号
- `#` 章节标题标记（改为纯文字标题）
- `##` 子标题标记
- `**` 粗体标记
- `- ` 列表符号（改为纯文字条目，不用列表前缀）
- `- **` 组合符号
- `🔴` `🔵` `✅` 等装饰符号
- `😊` `😟` `🎉` 等表情符号

### system prompt中的"# 角色设定" → "角色设定"
### system prompt中的"# 输出格式" → "输出格式"
### system prompt中的"# 创作原则" → "创作原则"
### system prompt中的"# 画面铁律" → "画面铁律"
### system prompt中的"# 核心叙事" → "核心叙事"

**原因**：AI在输出时会模仿system prompt中的符号格式，导致输出内容包含无用符号。

### patch工具避坑
- 不要在被替换文本中包含 `\"` 转义——会导致 `Escape-drift detected`
- old_string不要太长，使用足够唯一的短片段
- 当 `patch` 多次失败时，改用 `sed -i` 精确单行替换

### 修改时的关键经验

1. **永远不要只修一处** — 用户发现一处bug，一定有其他地方有同样问题。先grep穷举所有同类位置再修
2. **总纲硬编码和_build_child_*的system prompt是独立的两处** — 修改模式输出格式时，可能总纲header和_build_xxx方法都要改
3. **符号投诉是最高优先级信号** — 格林主人对符号标记（**、-、1、2、3、→、#）零容忍，遗漏任何一处都会引发暴怒
4. **patch工具的转义陷阱** — 多行Python字符串的`\\n`转义会被patch误判。new_string/old_string不要包含`\\\"`序列。优先用Python脚本整体替换
