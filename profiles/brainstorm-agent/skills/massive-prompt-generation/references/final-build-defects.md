# final_build.py 已知缺陷清单 (2026-05-27 审计)

## 1. 分类逻辑：硬编码关键词匹配，非语义理解

`classify_sentence()` 先用 `tech_markers` 检测是否为"技术段"（`摄影`+`景深`+`对焦`≥2个），然后按关键词命中顺序粗放分类。

**表现**: D03_材质质感 10万条中大量内容实际描述的是"光照条件/摄影风格"，C03_色彩调性只有449条（因为被D06先抢走了）

**影响**: 维度库内容混杂，组合时从某维度抽取的内容可能实际是另一个维度的描述

## 2. 过滤器粗放

`should_exclude()` 只检查：
- 前80字符匹配排除词（美甲/指甲/甜品/产品摄影/多人群组）
- 前150字符无人物关键词
- 完全没有检查：比喻词(仿佛/犹如/就像)、科技词(量子/粒子/全息/赛博)、三手描述、服装堆叠、场景矛盾(室内+户外混搭)

## 3. 维度条目是整句拷贝，非精确片段

每条内容是 `[维度名]源文件中的一个完整句子`，不是精准拆解后的原子片段。
- D03_材质质感条目例子: `柔和自然光，暖色调，浅景深，聚焦于女子面部和手部` — 这说的是光照/构图，不是材质
- 组合时如果直接拼接这些"整句"会重复大量冗余信息

## 4. 三个维度严重不足

| 维度 | 条目数 | 问题 |
|:----|:-----:|:----|
| C01_美学风格 | 840 | 条目太少，组合时重复率高 |
| C03_色彩调性 | 449 | 条目太少，多数内容被D06先抢走了 |
| C04_构图镜头 | 2,635 | 偏少，且内容混杂 |

## 5. 无组合引擎脚本

有建库脚本(final_build.py)、有模板文档(01_模板结构.txt)、有变体引擎(variant_engine.py)、有gen_prompts.py(硬编码模板)。
**但没有脚本能从17维库各抽1条→按组合规则拼接→润色→输出完整prompt。**
