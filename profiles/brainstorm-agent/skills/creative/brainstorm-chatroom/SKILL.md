# 头脑风暴聊天室

## 何时使用
当需要深度理解一篇文章、观点、论文、想法时使用。适合在以下场景：
- 读了一篇好文章想深度理解
- 有一个模糊的想法需要多视角审视
- 判断一个方向值不值得做
- 想从多维度评估一个方案

## 六个思维角色

### 1. 系统架构师
关注：观点如何变成可运行的系统
- 输入是什么？输出是什么？
- 中间状态如何表示？
- 哪些模块必须有？哪些可以后置？
- 最小可验证版本是什么？
- 系统边界在哪里？

### 2. 产品怀疑者
关注：是否真的有用户价值
- 这个问题是否真实存在？
- 用户是否愿意付出时间/金钱/迁移成本？
- 是否只是技术自嗨？
- 是否存在更简单的替代方案？
- 落地时最可能死在哪里？

### 3. 投资人视角
关注：长期壁垒和商业化空间
- 这个方向的天花板在哪里？
- 竞争对手会怎么做？
- 护城河是什么？（技术/数据/网络效应）
- 变现路径是否清晰？
- 时机对还是不对？

### 4. 哲学观察者
关注：底层假设和认知盲点
- 这个观点的隐性假设是什么？
- 类比是否合适？
- 是否存在确认偏误？
- 如果完全错了，最可能的原因是什么？
- 5年后是否还有意义？

### 5. 写作编辑
关注：信息表达和读者接收
- 能一句话说清楚吗？
- 目标读者是谁？
- 信息密度够不够？
- 如果写成文章，最好的切入点是什么？

### 6. 反方辩手
关注：最致命的弱点
- 最好的反例是什么？
- 什么条件下会完全失效？
- 支持者最容易忽略的风险是什么？
- 失败模式是什么？
- 有没有更简单的替代解释？

## 讨论流程

### 第一轮：核心价值
每个角色用150字以内说出自己看到的核心价值

### 第二轮：最大问题
每个角色指出一个最大问题

### 第三轮：互相回应
角色之间互相回应对方的观点

### 第四轮：主持人总结
主持人结构化总结，包含：
1. 真正有价值的地方
2. 最大的不确定性
3. 对我的具体启发
4. 是否值得写成公众号文章（附理由）
5. 是否值得沉淀成Skill/Memory/Eval（附理由）

## 输出保存
- 完整讨论总结 → outputs/ brainstorm-topic-YYYY-MM-DD.md
- 知识卡片 → memory-cards/card-YYYY-MM-DD.md
- 长期结论（仅1-3条稳定结论） → 写入Memory
