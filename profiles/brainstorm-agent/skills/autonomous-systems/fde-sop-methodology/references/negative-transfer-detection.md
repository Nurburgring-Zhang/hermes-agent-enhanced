# Negative Transfer Detection for Hermes Skills

Based on Microsoft Research paper **From Raw Experience to Skill Consumption: A Systematic Study of Model-Generated Agent Skills** (arXiv:2605.23899, May 2026).

## Key Findings

- **75% of model-generated skills are beneficial** on average
- **25% cause negative transfer** — using the skill makes performance WORSE than no-skill baseline
- Extractors and consumers are **asymmetric**: a model can be a strong extractor but weak consumer
- **Skill utility is independent of model scale or task baseline strength**

### 2. A/B Test Quick Check

When reviewing a specific skill:
1. Load the skill content with `skill_view()`
2. Manually perform the task WITHOUT loading the skill — note quality/difficulty
3. Load the skill and repeat — compare
4. If the skill version is WORSE → flag for negative transfer

### 3. Semi-Automated Detection

Run periodically (can be delegated via cron):


### 4. Known Risk Profiles

| Skill Type | Negative Transfer Risk | Why |
|---|---|---|
| Very narrow (single PR/issue) | HIGH | Overfits to one scenario |
| Very long (>500 lines) | MEDIUM | Contains conflicting advice |
| Generated by LLM one-shot | MEDIUM (25%) | No validation gate |
| Frequently patched | LOW | Continuously validated |
| Human-crafted for class | LOWEST | Best baseline |

## Mitigation

1. **Validation gate** before any patch: measure, accept only if improvement
2. **Rejection buffer**: keep rejected edits as negative examples
3. **Periodic audit**: flag skills with declining success rates
4. **Consolidation**: merge overlapping narrow skills into class-level skills
