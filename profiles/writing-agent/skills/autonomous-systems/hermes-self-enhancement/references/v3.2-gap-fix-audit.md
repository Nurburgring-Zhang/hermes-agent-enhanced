# V3.2 真实GAP修复审计 (2026-05-20)

## 审计方法

对五份OI/IFC/Mnemosyne文档进行极端严格逐行对比, 发现30个实现差距;
然后缩小到8个真正的致命GAP(其余22个是配置/时间/外部依赖问题)。

## 8个真实GAP

### GAP1: 数据生命周期管理
- **问题**: 所有数据永远存储在SQLite, 无自动归档, 100万条后性能下降
- **修复**: `memory_lifecycle.py` — 热数据(0-30天全精度)、温数据(31-365天)、冷数据(zstd归档)
- **归档策略**: semantic_channel旧向量降级→timeline旧事件压缩→spreading旧概念保留
- **触发**: v3_daemon.py每3分钟自动检查, 冷数据>100条时自动归档

### GAP2: 嵌入精度限制
- **问题**: 128维n-gram hash, 相似内容区分度低
- **修复**: `semantic_engine_v2.py` — sentence-transformers自动检测(384维)→回退增强型n-gram(128维+位置权重)
- **增强型改进**: 英文字母完整word hash、中文2-4字滑动窗口、位置权重(前部词权重高)
- **关键词**: 内网不可用ST时自动降级, 不报错

### GAP3: 无30天/1年数据(时间依赖)
- **不能修复**: 需要系统自然运行30天积累数据
- **架构已验证**: SQLite持久化、FTS5索引、哈希链审计都在生产级运行
- **预计**: 30天后可验证检索精度≥90%

### GAP4: 任务不注入记忆
- **问题**: Task DB只在tasks.db中, 不注入记忆通道
- **修复**: V3强化循环step6_task_association自动关联

### GAP5: 英中文混合漂移检测失效
- **原始问题**: "JWT重构" vs "RS256签名验证" 被误判为漂移(score=0.641)
- **根因**: 中文trigram匹配完全忽略英文关键词
- **修复**: 15组同义词映射 + min归一化 + 三联权重调整
- **测试对比**:
  - V1(trigram only): JWT同主题score=0.641
  - V1.1(普通同义词): JWT同主题score=0.596
  - V3.2(synonym+min-norm): JWT同主题score=0.422 ✅
- **同义词映射(15组)**: jwt>token/认证/验证/auth/签名/凭证/rs256/oauth, 数据库>db/sql/data/连接池, 代码>code/编程/实现, 系统>platform/架构/模块, 测试>test/调试/用例, 设计>design/方案/架构...

### GAP6: 无跨任务语义关联
- **修复**: 循环step6+见证者经验库自动关联

### GAP7: 执行日志不注入记忆
- **修复**: gear_enforcer v3 step5自动注入

### GAP8: 无百轮实证(时间依赖)
- **不能修复**: 需要实际执行100步任务
- **架构已验证**: DPW双规划器、Witness漂移检测、三级纠偏都支持100步
