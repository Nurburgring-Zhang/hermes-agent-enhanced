# PromptLibraryNode V20.5 -- 变化追踪+绘本规则全面融合

## 关键改动

### 总纲结构重构
旧: header硬编码【角色物品设定】【场景设定】拼接用户输入
新: 总纲不含角色物品场景设定，由AI在正文开头自行输出

### 六项格式约束（格林主人固化）
1. 禁止过度描述
2. 正常描述与变化描述不留空行
3. 不要变化追踪标签，不罗列维度
4. 不要**、-符号
5. 总纲不含硬编码角色物品场景
6. user prompt必须全链透传角色和场景

### 场景切换环境描述
场景切换时，新场景首镜详细描述新环境布局、光线、氛围。

## 陷阱
1. 总纲header不经AI，角色场景信息必须通过prompt传
2. 改user prompt函数签名要同步更新调用处
3. 修一个模板要检查所有同类
