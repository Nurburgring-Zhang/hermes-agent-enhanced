# 2026-06-01 最终规则 — PromptLibraryNode V20.5+ 

## 会话概要
格林主人对PromptLibraryNode输出格式进行了多轮（至少4轮）严格修正。核心问题：
1. 变化标注逻辑错误（输出不需要的维度和"无"值）
2. 总纲缩进错误（出现4空格、8空格）
3. 输出中包含`#` `##` `**` `- ` `🔴` `😊`等符号
4. 对修正响应不够全面（只修一处不修全部）

## 最终规则（2026-06-01 格林主人确认的最终要求）

### 一、总纲格式
7个编号项，用中文顿号（1、2、3...）
内容行零缩进，不能有任何前置空格
角色物品设定多角色时独立行，末尾不加句号
场景设定行末尾不能自动加句号（用户输入带句号即可）

### 二、分镜头字段
始终输出：景别、画面、运镜、转场、时长
仅在变化时输出（无变化时整行消失，包括标题）：分镜场景、角色特征
仅在必要时输出：叙事功能

### 三、变化标注 — 最终版
**只追踪场景和角色两个维度的变化**
有场景变化 → 分镜场景行：完整场景描述（2-4句）
有角色变化 → 角色特征行：完整角色描述（2-3句）
两者都变 → 各一行
无变化 → 两行整行不出现，不写任何相关字样
其他维度（服装/道具/时间/情绪等）不追踪

### 四、符号禁令（绝对强制执行）
system prompt中禁止所有：`#` `##` `**` `- ` `- **` `🔴` `😊` `📦` `☀️` `🔄` `👤` `👗`
system prompt标题用纯文字，不用标记
每个system prompt末尾加入：`重要：输出中不要包含任何** - 等符号标记，不要用星号或横线装饰文字。直接输出纯文字。`

### 五、修正响应模式
当用户对格式问题发火时（使用激烈语言如操你妈、弱智等），意味着存在多个同类问题。
正确响应：穷尽式搜索所有模式 → 全部修正 → grep验证无残留 → API测试 → 汇报
