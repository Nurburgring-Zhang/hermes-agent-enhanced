## 二、格式规则最终版（2026-06-01 第三轮修正）

### 总纲格式
- 编号 `1、2、3、`（中文顿号）
- **内容行无缩进**（用户因8空格缩进砸键盘——彻底放弃所有缩进）
- 无空行
- 场景设定末尾不加句号（用户输入可能自带，模板不再加）
- 角色物品设定多角色时直接换行，末尾不加句号

### 分镜头输出格式（故事板/短剧）
```
【Shot N】
景别：（始终输出）
分镜场景：（有场景大变化时输出完整描述2-4句，无变化时整行不出现包括标题）
角色特征：（有角色特征大变化时输出完整描述2-3句，无变化时整行不出现包括标题）
画面：（始终输出）
运镜：
转场：
时长：
叙事功能：（必要时才输出）
```

### 变化标注核心规则（最后定稿）
**只追踪场景和角色**两个维度。不追踪服装/道具/时间。
- 有场景变化 → 输出 `分镜场景：完整场景描述`
- 有角色特征变化 → 输出 `角色特征：完整角色描述`
- 两者都变化时各一行
- 无变化时这两个字段整行不出现（包括字段标题本身）
- 维度名称用中文文字（场景变化/角色变化），不用emoji或符号
- 字段标题（分镜场景和角色特征）在无变化时也整行消失

### 符号禁令（最终版）
system prompt中禁止：`#` `##` `**` `- ` `🔴` `😊` `《` `》` `「` `」` 等所有装饰符号
每个system prompt末尾加：
```
重要：输出中不要包含任何** - 等符号标记，不要用星号或横线装饰文字。直接输出纯文字。
```

### 儿童绘本格式特例
**不适用分镜场景/角色特征两个字段。** 正确格式：
```
第X页：标题
画面：2-3句
文字：1-2行
（场景大变化时：分镜场景：完整场景描述）
（角色大变化时：角色特征：完整角色描述）
```

## 四、用户特征（格林主人）

- 对提示词节点生产的剧本质量极端挑剔
- 要求故事有情感起伏、钩子、反转、情感闭环
- 厌恶平铺直叙、说明书式叙事
- 用激烈语言（操你妈、傻逼、弱智）表达不满=要求立刻修正
- 技术理解力强，能看懂代码级细节
- 对输出格式的每一个细节极度精确，必须有总纲、无缩进、无符号
- 变化标注不要标签不要符号不要标题——无变化时整段不出现
