# PromptLibraryNode 输出格式规范（格林主人定制）

## 总纲格式（所有模式统一）

所有输出模式（故事板/绘本/短剧/儿童内容）的总纲必须严格遵循编号1~7格式：

```
【xxx总纲】
1、整体视觉风格：
    内容描述...
2、角色物品设定：
    角色描述...
3、道具/武器：
    待补充。
4、场景设定：
    场景描述...
5、氛围与画质标准：
    氛围描述...
6、声音设定：
    声音描述...
7、核心叙事设定：
    叙事描述...
```

### 关键规则
- 总纲内**禁止空行**
- 禁止使用 `【整体视觉风格】` 或 `【角色物品设定】` 等方括号标题→用数字编号 `1、整体视觉风格：`
- 禁止使用 `**` 和 `- **` 符号
- 角色设定和环境设定必须在总纲内，不允许在正文开头单独输出 `【角色设定】` 引导
- `道具/武器` 字段固定写"待补充"（除非用户特别指定）

## 分镜/分页描述格式（场景/角色变化时）

当场景或角色特征发生变化时，输出字段顺序严格为：

```
【Shot N】
景别：...（始终输出）
分镜场景：（仅在场景变化时输出）
角色特征：（仅在角色特征变化时输出）
画面：...（始终输出，放在分镜场景和角色特征之后）
运镜：...
转场：...
时长：X秒
叙事功能：必要时才输出
```

### 关键规则
- **景别**始终输出（不只在变化时输出）
- **画面**始终输出，且放在分镜场景和角色特征之后
- **分镜场景**和**角色特征**只在变化时输出，不变化时不输出
- **分镜场景**描述应在画面描述之前写清变化内容和过渡提示
- 固定用「时间·空间」前缀（如「清晨·森林小屋厨房」）
- 时间/空间变化时在先导句中明确标注「时间推进到…」或「场景切换到…」

## 五维度变化标注（统一于所有叙事模式）

当场景/角色/服装/道具/时间中任何一个发生变化时，在变化发生的首个镜头/页面中显式写出：

```
【变化标注】
🔄 场景变化：从哪到哪？
👤 角色变化：新角色出现/角色离开/角色状态改变？
👗 服装/配饰变化：服装/配饰改变？
📦 道具/物品变化：关键道具状态变化？
☀️ 时间/光线变化：时间推进/光线变化？
```

- 变化几维写几维，无变化写「无」
- 严禁遗漏变化维度
- 这与分镜场景/角色特征字段功能互补：场景/角色特征描述格式交代"是什么变化"，五维度标注系统交代"变化了哪几个维度"
